Smart IoT-Based Patient Health Monitoring & Cyber-Attack Detection System
Overview

This project presents a secure IoT-based healthcare monitoring system designed to collect and analyze patient health data in real time while protecting it from cyber threats. The system integrates IoT devices, cloud computing, cybersecurity techniques, and blockchain-based validation to ensure that medical data remains accurate, trustworthy, and secure.

Modern healthcare systems increasingly rely on wireless IoT devices for patient monitoring. While these technologies improve efficiency and response time, they also introduce new security risks that can compromise sensitive medical information.

Problem Statement

Hospitals and healthcare facilities are rapidly adopting wireless IoT technologies to monitor patient health data in real time. However, these systems are vulnerable to several cyber threats, including:

Data manipulation

Spoofing attacks

Replay attacks

Unauthorized access

If attackers inject false or altered health data, doctors may receive incorrect information about a patient's condition. This can lead to wrong diagnoses, delayed treatment, and serious risks to patient safety. Most existing monitoring systems focus only on data visualization and lack strong mechanisms for data integrity verification and cyber-attack detection.

Proposed Solution

This project proposes a secure and intelligent IoT healthcare monitoring framework that ensures both real-time monitoring and cyber-attack detection.

Patient health data is collected from IoT-based bedside monitoring devices and transmitted to a cloud server where it is processed and visualized through a live web dashboard. To protect sensitive medical information during transmission and storage, the system applies encryption, hashing, and digital signature techniques.

A lightweight TinyML-based edge intelligence model analyzes incoming data patterns to distinguish between genuine medical emergencies and suspicious or manipulated data generated by attackers. The backend continuously evaluates system activity and assigns a trust status indicating whether the system is operating safely or under attack.

To ensure long-term data integrity, verified health data hashes and security logs are stored using a consortium blockchain mechanism, allowing trusted healthcare entities to validate records and detect any tampering attempts.

If a cyber attack occurs, the system immediately identifies the event and displays key details such as the attacker’s IP address, attack type, and timestamp on the monitoring dashboard.

Key Features

Real-time IoT-based patient health monitoring

Secure data transmission using cryptographic techniques

Cyber-attack detection and logging

TinyML-based anomaly detection for health data

Blockchain-based verification of medical data integrity

Live dashboard displaying patient health status and system security alerts
